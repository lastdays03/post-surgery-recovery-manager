# 🔮 [미래 비전] 하이브리드 AI 온보딩 및 정밀 의학 프로파일링 계획

본 문서는 진단서, 소견서 등 의료 문서를 AI로 분석하는 **OCR 기술**과 사용자와의 **대화형 인터페이스(Chat)**를 결합하여, 사용자의 건강 상태를 가장 정밀하게 파악하고 최적의 회복 식단을 자동화하는 하이브리드 고도화 계획을 정의합니다.

## 1. 정밀 프로파일링 핵심 데이터 필드
정밀한 식단 추천을 위해 수집해야 할 의학적 지표의 기준과 의미를 정의합니다.

| 구분 | 데이터 필드 | 기준 및 의학적 의미 |
| :--- | :--- | :--- |
| **수술 정보** | 수술 종류 | GI(위·대장·췌장), 정형, 두경부, 이식, 비만수술 등 (프로토콜 결정) |
| | 수술 날짜 | D-Day 기준 회복 및 식단 단계 계산의 기점 |
| | 수술 범위/침습도 | 대수술 vs 경수술 (대사 스트레스 강도 판단) |
| **영양 위험도** | NRS-2002 점수 | ≥3점 위험, ≥5점 고위험 (영양 집중 지원 필요성 판단) |
| | 체중 변화 | 6개월 내 10~15% 감소 시 중증 위험 (영양 불량 상태) |

| | SGA 등급 | C등급일 경우 중증 영양 불량으로 간주 |

| **대사/질환** | 기저질환 | 당뇨, 고혈압, 간·신장 질환 (탄수/단백·전해질 조절의 근거) |
| | 소화기 기능 | 위배출 지연, GERD (탄수화물 음료 및 식단 종류 제한) |
| **섭취 능력** | 경구 가능 여부 | 입으로 음식 섭취가 가능한지(Yes/No) 여부 |
| | 예상 금식 기간 | >5일, >7일, >14일 (영양 공급 경로 및 강도 결정) |
| | 섭취율 | 필요량의 50% 미만 섭취 여부 (추가 보충제 필요성) |
| **연령/근감소** | 연령 | 고령자일 경우 더 높은 단백질 권장량 적용 |
| | 근실실(Sarcopenia) | 근감소증 존재 시 합병증 위험 증가 및 재활 강도 조정 |

## 2. 하이브리드 온보딩 워크플로우 (Hybrid Onboarding)
문서 분석의 효율성과 채팅의 유연성을 결합한 하이브리드 구성을 지향합니다.

### A. 유형 1: 진단서 기반 온보딩 (OCR + Chat Supplement)
문서를 우선적으로 활용하여 입력 피로도를 최소화하고 정확도를 높입니다.
1. **문서 업로드**: 사용자가 진단서, 소견서, 퇴원 요약지를 촬영하거나 업로드합니다.
2. **AI 데이터 추출(OCR)**: Google Document AI 등을 통해 위 테이블의 핵심 지표를 자동 추출합니다.
3. **부족 데이터 보충(Chat)**: AI가 추출하지 못한 정보(예: 현재 식사율, 알러지 등)나 모호한 부분을 채팅을 통해 사용자에게 추가 질문합니다.
4. **최종 승인**: 추출 및 수집된 정보를 요약하여 보여주고 사용자가 정정/승인합니다.

### B. 유형 2: 순수 채팅 기반 온보딩 (Pure Chat)
진단서가 없거나 문서를 원치 않는 경우 기존 대화형 인터페이스를 통해 정보를 수집합니다.
- **스마트 가이드**: 사용자가 수술명만 말해도 AI가 예상되는 위험 요소를 먼저 제시하며 질문을 이어갑니다.
- **적응형 질문**: 앞에서 수집된 정보를 바탕으로 불필요한 질문은 건너뛰고 핵심 데이터 필드를 채워나갑니다.

## 3. 핵심 기술 및 데이터 거버넌스
### A. 데이터 정규화 및 알러지 관리
- **유연한 기저질환 처리**: 정의된 주요 질환 외에는 원문 그대로 보존하여 식단 AI에 전달합니다.
- **알러지 필드 독립화**: 안전을 위해 `allergies: string[]` 필드를 별도로 관리하여 식단 필터링의 최우선 순위로 둡니다.

### B. 보안 및 개인정보 보호
- **PHI 암호화**: 민감한 의료 데이터는 전송 및 저장 전 과정에서 암호화됩니다.
- **비식별화**: AI 모델 분석 시 이름, 연락처 등 식별 정보를 제거하는 데이터 마스킹을 수행합니다.
