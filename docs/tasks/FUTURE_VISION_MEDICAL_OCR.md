# 🔮 [미래 비전] AI 진단서 분석 기반 초개인화 온보딩 계획

본 문서는 장기적인 관점에서 진단서, 소견서 등 의료 문서를 AI로 분석하여 환자 맞춤형 식단 추천을 자동화하는 고도화 계획을 담고 있습니다.

## 1. OCR 기반 의료 문서 디지털화
사용자가 촬영하거나 업로드한 진단서를 텍스트로 전환합니다.
- **기술 스택**: Google Document AI, AWS Textract 혹은 OCR 전용 LLM 모델 활용.
- **핵심 추출 항목**: 수술명, 수술 일자, 기저질환, 알러지 정보, 담당의 주의사항(예: 저당 식단 필수, 단백질 제한 등).

## 2. 의학적 컨텍스트 분석 및 프로파일링
단순 텍스트를 넘어 환자의 상태를 의학적으로 이해합니다.
- **의학 용어 매핑**: 복잡한 의학 용어를 시스템 표준 수술 타입(`SurgeryType`) 및 회복 프로토콜로 자동 매핑.
- **위험 요소 자동 감지**: 진단서상의 수치를 분석하여 식단 생성 시 반드시 지켜야 할 'Red Flag' 재료 자동 설정.

## 3. 완전 자동화 온보딩 흐름
사용자가 질문에 답하는 대신 문서 한 장으로 온보딩을 완료합니다.
- **문서 업로드 → AI 분석 → 정보 확인(UI) → 식단 생성**으로 이어지는 원스탑 프로세스.
- 사용자는 AI가 추출한 데이터가 맞는지 최종 확인만 진행.

## 4. 의료 데이터 보안 및 개인정보 보호
민감한 의료 정보를 다루기 위한 보안 체계를 구축합니다.
- **데이터 암호화**: 전송 및 저장 시 개인 건강 정보(PHI) 암호화.
- **비식별화 처리**: AI 모델 학습 및 분석 시 개인 식별 정보를 제거하는 로직 구현.
